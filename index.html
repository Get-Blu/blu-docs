The AI agent that lives in your editor

blu is a high-performance VS Code extension that understands your entire codebase. Not just a chat—a partner that writes, debugs, and refactors in real-time.

How blu works

Step 01
Context Discovery

blu scans your project structure and local files to build a temporary semantic index. It knows where your components, types, and utilities live.

Step 02
Agentic Reasoning

When you ask a question, blu doesn't just guess. It plans a multi-step execution—searching code, reading files, and testing assumptions.

Step 03
Direct Implementation

Once the plan is ready, blu writes the code directly into your editor. You can review, undo, or iterate with a single click.

Frequently Asked Questions

Everything you need to know about blu.

What is blu and how is it different from Other AI tools? +
blu is a fork of Cline, optimized for performance and extended with additional model support. While we maintain compatibility with Cline's core features, blu adds faster workspace indexing, multi-model flexibility, and enhanced terminal integration. We're grateful to the Cline community for creating the foundation we build upon.
Do I need to pay for API access separately? +
Yes. blu is free and open-source, but you'll need to provide your own API keys from providers like Anthropic, OpenAI, or run models locally via Ollama. This gives you full control over costs and data privacy. Pro and Team plans include priority access to certain models.
Can blu work completely offline? +
Absolutely. Connect blu to Ollama and run models like Llama 3 or Mistral entirely on your local machine. No internet required, no data leaves your device. Perfect for working with sensitive codebases or in air-gapped environments.
How does workspace indexing work? +
When you open a project, blu scans your file structure and creates a semantic map using AST parsing. This index stays local on your machine and helps blu understand relationships between files, imports, and function definitions. It's rebuilt automatically when files change.
Is my code used to train AI models? +
Never. We don't store or transmit your code to our servers. When using cloud providers, your code is sent directly to them via your API keys. Check each provider's data retention policy—most offer zero-retention options for enterprise users.
What languages and frameworks does blu support? +
blu works with any language VS Code supports. We've optimized context understanding for JavaScript, TypeScript, Python, Go, Rust, Java, and C++.
Can I use blu with my team? +
Yes. The Team plan includes shared context libraries, allowing your team to maintain consistent coding patterns and reusable prompts. Each developer still uses their own API keys, maintaining individual cost control and privacy boundaries.
How do I contribute to blu's development? +
We welcome contributions! Visit our GitHub repository to submit issues, feature requests, or pull requests. As an open-source project, blu thrives on community input. Check our contributing guidelines for code standards and development setup.
